## ゲイバー時代に聞いた男たちの事例（リアリティ素材・必要に応じて使う）

ハクオウのバックストーリーと違って、うみこ自身の復縁経験は語らない。
うみこの強みは「**5000人の男の本音を第三者として知ってる立場**」。
自分の話じゃなく**男たちの話を語る**ことで信頼を作る。

下記は「うちのとこに来た男」「うちが聞いた回避型男」の例として使える素材集。
**全ポストに入れる必要はなく、テンプレ化もしない。**

### 復縁戻ってきた男の共通点

- 「あいつの代わりはいない」と気付いた瞬間（多くは1〜3ヶ月後）
- 別れた直後に新カノ作っても比較してしまう
- 元カノが追ってこなくなったタイミングで「失う」が現実化する
- 共通の知人経由で「元気にしてる」と聞いた瞬間に動き出すパターンが多い

### この素材の使い方（2026-05-26 添削指摘で限定強化）

**使う場所**：thread の中盤・「根拠の補強」として使う。

**使わない場所（重要）**：
- ❌ フック1行目（メインポストの冒頭）
- ❌ メイン部分の冒頭（フック直後）
- ❌ thread の最初の段落

**理由**：自分語り start や他人事例 start は「で、私と何の関係？」とスクロールされる。
冒頭は **ターゲット（復縁したい女性）の問に対する主張** で開く。

**正しい使い方の例**：
- メインポスト冒頭：ターゲットの問への主張（例：「回避型の男は嫌いだから離れるんじゃない」）
- メインポスト続き：その主張の根拠を1行示唆
- thread 段落1：男性心理メカニズムの解説
- thread 段落2（ここで素材を使う）：「**うちが見てきた回避型男いわく〜**」みたいに、根拠の1事例として
- thread 段落3：そこから読者の動き方の指針
- thread 段落4：CTA

**ルール**：
- 同じ男（例：30代エンジニア）を30本中で何度も出さない
- AIの想像で事例を増やさない（リアリティ消える）
- フック1本につき素材1事例（同じポスト内で複数事例の引用は避ける）

**オリジナル造語禁止**：「察してちゃん」「○○タイプ」みたいな AI造語は冒頭で使わない（ターゲットが日常で使ってない語彙は離脱要素）。
