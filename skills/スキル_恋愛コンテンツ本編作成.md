# スキル_恋愛コンテンツ本編作成

note有料部分の本編コンテンツを書くためのスキル。
レター（スキル_恋愛レター作成.md）とトーンが違うので、別スキルとして独立。

---

## このスキルとレターの差

| 項目 | レター | 本編（このスキル） |
|---|---|---|
| 主目的 | 感情訴求で購入決断を作る | 教育・実践情報を提供する |
| 文体ベース | 全文キャラ口調 | **章全体で一貫したキャラ口調**（敬語と混在NG） |
| キャラ口調 | 全文「〜だよね」「〜なんだよね」 | 全文に散らす（薄めるが切らない） |
| 絵文字 | 多用（🥲😭🥺） | レターの1/3程度／章の節目のみ |
| 体験談 | 防衛解除フェーズで集中投下 | 各章のリードと実例で散らす |
| 自虐 | 全体に散らす | 章のリード部のみ |
| ロジック比率 | 30%／感情70% | ロジック70%／感情30% |

### ★最重要：文体の一貫性（v2追加）

**章の途中で「キャラ口調」→「敬語」に切り替えるのは絶対NG**。
ここがAI感の最大の原因。

例えば経験者女性キャラなら、章全体を通して：
- ❌ リード「〜だよね」 → 教育「〜です／〜ます」 → まとめ「〜だよね」
- ✓ リード「〜だよね」 → 教育「〜なんだよね」「〜だったりする」 → まとめ「〜だよね」

教育パートでも、語尾の50%以上をキャラ口調で維持する。
「説明調」と「キャラ調」を**切り替えるんじゃなく、混ぜる**。

## 本編の章構成テンプレ

各章は以下の4ブロック構成で書く：

### ブロック1：リード（キャラの体験談・自虐込み）

- 私の失敗エピソード or 当事者目線の導入
- 「私もここで失敗してた」を1〜2段落
- 絵文字1〜2個
- キャラ口調OK

### ブロック2：本編・教育パート

- 普通の教育文体（です／ます or 中立言い切り）
- ロジック軸：メカニズム／ステップ／注意点を整理
- 抽象→具体の順で展開
- キャラ口調**最小限**（基本中立）

### ブロック3：実例パート

- 「実際にこのやり方でこうなった」
- 自分の彼の本音セリフ or 相談者事例
- 2〜3個並べる
- 絵文字なし or 控えめ

### ブロック4：まとめ（キャラ口調で締め）

- ここの内容の核を1〜2段落
- キャラ口調で短く
- 絵文字1個
- 次章への橋渡し

```
[レター冒頭]

↓
[本編リード]
キャラ口調50%・絵文字1〜2個・自虐1個
↓
[本編教育パート]
中立文体・絵文字なし・ロジック中心
↓
[本編実例]
中立文体・絵文字なし・セリフ引用中心
↓
[本編まとめ]
```

教育パートは**普通の本のように**読める文体にする。
キャラ口調を全文に効かせると「ノウハウじゃなくおしゃべり」になり、読者が学ぶ姿勢で読めなくなる。

## キャラ別トーン調整（章全体で統一）

### 経験者女性キャラ（あや・もも・みるく系）
- **章全体で**：「〜だよね」「〜なんだよね」「〜らしい」「〜だったりする」基調
- 教育パートも「〜なんだよね」混じりで書く（「〜です」に倒さない）
- 絵文字：🥲（最多）／😭／🥺（章の節目のみ）
- 口語強調OK：「マジで」「ぶっちゃけ」「全然」「めっちゃ」

### 男性当事者キャラ（凪・ななせまる・はる系）
- **章全体で**：「〜なんですよね」「〜だな」「〜と思う」「〜だったりする」基調
- 教育パートでも口調をキープ
- 絵文字：基本不使用

### 専門家系（ハクオウ系）
- **章全体で**：「〜だ」「〜と思う」「〜してる」「〜らしい」基調
- 「〜である」「〜にもかかわらず」みたいな論文調はNG
- 絵文字：不使用
- 「臨床心理学的に」等の専門ワードは要所のみ

## 過剰NG（v2追加・AI感の典型）

以下は全部禁止。出てきたら書き直し：

### 過剰見出し
- 章の中で「**○○**」太字見出しを連発する
- セクション内に「**実際の本音**」「**理由1**」「**ポイント**」みたいな小見出しを多発する
- → 自然な文章の流れの中に溶かす。見出しは章タイトルと節タイトルのみ

### 過剰括弧／太字
- 強調したい箇所を全部太字にする
- 1ページに太字が10個以上ある
- → 太字は1セクションに1〜2個まで、本当に強調したい箇所のみ

### 過剰箇条書き
- 説明文を全部箇条書きに変換する
- リード／本文／まとめ全部箇条書き
- → 箇条書きは「並列が複数ある時」のみ。流れがある説明は文章で書く

### 過剰絵文字
- 1段落に絵文字2個以上
- 文章中の感情語にいちいち絵文字
- → 絵文字は章の節目のみ、1段落に1個まで

## note貼り付け用フォーマット（必須・AIキット_note貼り付けルール準拠）

最終的にnoteに貼り付ける前提で書く。**noteで壊れる記法は最初から使わない**。

### ✅ 使っていい記法

- `## 大見出し`（章タイトル）
- `### 小見出し`（節タイトル、1-1, 1-2 など）
- `**太字**`（条件付き、下記参照）
- `> 引用`（グレー背景ボックス）
- 空行（段落区切り）

### ❌ 使ってはいけない記法

| 記法 | 代替 |
|---|---|
| `---` や `───` 区切り線 | 使わない。空行＋見出しで区切る |
| `- リスト` `* リスト` | `・` で通常テキスト |
| `1. 番号付きリスト` | `①②③` または「1.」（テキスト扱い） |
| `# h1見出し` | `##` を使う（noteの最大は##） |
| `#### h4以下` | `###` までにする |
| `[リンク](URL)` | URLを直書き |

### 太字の安全ルール

- 閉じ `**` の直後に全角文字を直接くっつけない
- 閉じ `**` の後には必ず半角スペース／句読点／改行を入れる
- 例：✅ `**太字** 。次の文` ／ ❌ `**太字**こと`

### 区切り線の代替

章や節の切り替えは **「空行2行＋次の見出し」** で表現。
`───` `---` のような区切り線は使わない。

## AI感を消すルール

口調文体ルール_一般人感.md と同じNG表現を避ける。
特に教育パートで以下が出やすいので注意：

| AI感が出る教育文体 | 自然な書き方 |
|---|---|
| 〇〇の構造を理解することが重要です | 〇〇の仕組みを知っておくのが大事 |
| アプローチを見直す必要があります | やり方を変える必要がある |
| ここでのレバレッジポイントは | ここで大事なのは |
| 認識を反転させることで | 見方を変えることで |
| 〇〇の本質に到達することで | 〇〇の本当の意味が分かれば |
| 圧倒的な差を生みます | かなり結果が変わる |
| 〇〇という事実を踏まえると | 〇〇を考えると |

教育パートでも**論文調にしない**。読者は学者じゃない。

## 章構成の例（あや・回避型復縁noteの第1章）

```
第1章：回避型の彼が離れた本当の理由

  1-1. 「もう疲れた」「重い」って言われた本当の意味
       → リード（私の体験）→ 教育（表面vs本音）→ 実例 → まとめ
  
  1-2. なぜ「冷却期間」「自分磨き」が逆効果なのか
       → リード → 教育（メカニズム）→ 実例 → まとめ
  
  1-3. 回避型と通常型の決定的な違い
       → リード → 教育（タイプ別）→ 実例 → まとめ
```

各セクションを1500〜2500字程度。
1章合計：5000〜8000字目安。
note全体（6章）：30,000〜50,000字目安。

## Phase 2：監視員チェック（書き終わった後に必ず実行）

### 第1章：AI感チェック【最重要】

AI感が残ってたらどんなに内容良くてもゴミ。
ターゲット（27〜29歳女性）は「知識量」より「自分を理解してくれてるか」を見る。

#### 語尾・リズム
- [ ] 同じ語尾が3回以上連続してない
- [ ] 短文と長文が混在してる
- [ ] 段落の閉じ方が「以上がポイントです」「次に進みましょう」みたいな定型じゃない
- [ ] 接続詞（さらに／また／したがって／そのため）が連続してない

#### スタンス・断定
- [ ] 「一概には言えませんが」「状況によって異なります」等の保険表現がない
- [ ] 「〜かもしれません」「〜と言われています」で逃げてない
- [ ] 「やめて」「やっちゃダメ」と言い切るべき箇所で曖昧にしてない
- [ ] 読者の味方に立ってはっきり意見を出してる

#### 構成の機械的パターン
- [ ] 「ここでは○○について解説します」のような前置き宣言がない
- [ ] 「ステップ1」「ポイント①②③」で全項目が同じ分量で並んでない（濃淡をつける）
- [ ] 4つ以上の項目を同じ構成で列挙してない
- [ ] 見出しで分かることを本文で再宣言してない
- [ ] 「結論から言うと」で始めてない（いきなり結論を書く）
- [ ] **★最重要：ナンバリング構造（パターン①②③／原則①②③／場面①②③）がコンテンツ全体で3箇所以上ない**（1〜2箇所まで、残りは番号なしの自然文）

#### 表現・比喩
- [ ] AI特有の比喩（地図／羅針盤／設計図／土台／柱／エンジン／スパイス／筋トレ／両輪／潤滑油）を使ってない
- [ ] 「革新的」「画期的」「必見」「本質」等の万能語を使ってない
- [ ] not A but B構文（「〜ではありません。〜です」）が3回以上ない
- [ ] 抽象語（「最適化」「価値を最大化」「本質を押さえる」）だけで押し切ってない
- [ ] 同義語の言い換え連打（重要／大切／欠かせない）をしてない

#### 濃淡・ムラ
- [ ] 全テクニック・項目が同じ文字数・構成・パターンになってない
- [ ] 重要テクニック（各章1〜2個）は500〜800字で深掘り、それ以外は100〜200字でサラッと
- [ ] 感情が入る箇所で文体が崩れてる（「いやこれマジで」「ほんとにね」等）
- [ ] 解説パターン（実体験／心理学／失敗例／勘違い／セリフ／問いかけ）がランダムに散ってる
- [ ] 熱い箇所とサラッと流す箇所の温度差がある

#### 記号・装飾
- [ ] emダッシュ／全角スラッシュ／→ が不自然に多用されてない
- [ ] 太字が過剰になってない
- [ ] `---` `───` 区切り線がない
- [ ] Markdownリスト記法（`-` `*` `1.`）が使われてない（noteでバグる、`・`で書く）

#### note貼り付け形式

**太字**：
- [ ] 小見出し（###）あたり**5〜6個**の太字が入ってる
- [ ] 太字の対象：核心の気づき／常識の逆転／結果／成果／決め台詞／共感ポイント／行動促進
- [ ] 太字なしのパラグラフが3つ以上連続してない
- [ ] **★絶対禁止：`**「`のように太字の直後に「を置かない**（noteで太字が効かなくなる）。`「**テキスト**」`のようにカッコの内側に太字
- [ ] HTMLコメント（`<!-- -->`）が残ってない

**引用ボックス（>）**：
- [ ] 小見出しあたり**3〜4個**の引用がアクセントとして入ってる
- [ ] 引用は必ず1行で完結（引用内改行禁止、noteでバグる）

**改行・段落**：
- [ ] 2〜3文で1段落（4文以上連続禁止）
- [ ] 1文ごとの改行でスカスカになってない
- [ ] 句点「。」が多すぎない（半分程度に減らしてテンポ）

#### 解説パターンの散らし【最重要】

以下9パターンがランダムに使われてるか。同じパターン3回以上連続NG：
- [ ] ① 作者の実体験（「私の場合は〜」）
- [ ] ② 男性心理の解説（「彼の頭の中では〜」）
- [ ] ③ 心理学・脳科学の理論
- [ ] ④ 失敗パターンの解説
- [ ] ⑤ よくある勘違いの指摘
- [ ] ⑥ 具体的セリフ例・LINE例文
- [ ] ⑦ 「なぜそうなるか」の深掘り
- [ ] ⑧ 読者への問いかけ
- [ ] ⑨ 相談者の事例（レター参照可）
- [ ] 全章通して最低5種類以上使ってる

#### 感情表現・臨場感
- [ ] 淡々とした教科書的文章が続いてない
- [ ] ノウハウの間に感情・物語・臨場感が挿入されてる
- [ ] 「心臓がバクバクした」「頭が真っ白になった」みたいな情景描写がある
- [ ] 「！」が効果的に使われててテンポある（過剰NG）

#### 読者への称賛パート
- [ ] 導入部分（購入直後の称賛）が入ってる
- [ ] 中盤（継続の称賛）が入ってる
- [ ] 結び（未来への称賛）が入ってる

- [ ] 「これで¥2,980なら安い」と感じる情報量

## 監視員ルール：B以下は書き直し

評価方法：
- A：完璧
- B：軽微な修正必要 → スポット修正
- **C以下：そのセクション全体のロジック・構成を見直して書き直す**

修正後のコンテンツ全文を完全な形で出力。
ノンストップで実行。途中でユーザー確認しない。
