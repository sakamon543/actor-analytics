# スキル_恋愛コンセプト設計

新アカウント立ち上げ時に、コンセプト・キャラ・プロフ・CTA・ストーリーを一気通貫で設計するスキル。

---

## 設計思想

**テンプレ埋め型じゃない。素材＋思考フロー型。**

- ナレッジ／素材を毎回読み込んで、AIが素材から思考する
- ショートカット（テンプレ穴埋めで一気に出す）はしない
- Web検索でリアル素材集めるのは**必須工程**（省略不可）
- 各Phaseで5案出し → 議論 → 1案確定の繰り返し

---

## Step 1：受講生インプット受領

受講生（クリエイター）から以下を受け取る：

```
■ ジャンル
・恋愛特化のサブテーマ（回避型復縁／不安型攻略／浮気／遠距離 等）
・想定価格帯（¥1,980 / ¥2,980 等）

■ キャラ希望
・タイプ（経験者女性／男性当事者／専門家／未定）
・参考にしたい既存アカウント（あれば）

■ 受講生の体験／武器（あれば）
・自分のリアル経験
・実績／資格／知識

■ ターゲット
・誰に届けたいか（年代／状況）
```

不足してる情報は受講生に確認する。曖昧なまま進めない。

## Step 2：キャラタイプ確定

### 3タイプの差別化

| タイプ | 武器 | 適合する人 | 例 |
|---|---|---|---|
| **経験者女性** | 自虐＋当事者語り、最強共感 | 自分が復縁経験ある女性 | もも・みるく・R系 |
| **男性当事者** | 男目線本音暴露 | 男性／振った側経験ある人 | 凪・ななせまる系 |
| **専門家** | データ・1000人見てきた断言 | カウンセラー／専門資格持ち | ハクオウ系 |

### 議論ガイド

受講生の素材を見て：
- 受講生に**実体験**があれば → 経験者女性 or 男性当事者
- 受講生が**知識中心**なら → 専門家
- 既存ハクオウ／あやと**カニバらない**かチェック

→ 1タイプ確定。

## Step 4：商品コンセプト確定（売れるコンセプト検査つき・2026-07-03強化）

### 思考フロー

1. AIキット_恋愛復縁_concept.md の5案・競合レター4本の商品構造を見直す
2. 受講生の素材／キャラタイプと一致するか吟味
3. **5案は「別々の切り口×別々の手法名」で出す**（同じ手法の言い換え5個は禁止。AIキット5案の派生だけで埋めない＝全アカが同じ顔になる）
4. 各案を下の**売れるコンセプト検査（4問）**に通してから提示 → 議論 → 1案確定
5. 確定したら `商品コンセプト台帳.md`（スキル直下）に1行追記

### 売れるコンセプト検査（5問・全部通らないと提示しない）

**判断基準の原典＝てっど理論**（`思考スキル_売上逆算.md` に結晶化済み。迷ったら理論①②原文へ）。
恋愛noteを買う人は「苦しんでる・答えを知りたい・安心したい」人。有料で買う理由はこの感情が動くかで決まる。

| # | 検査 | 判定基準（原典） |
|---|---|---|
| 1 | **ベネフィット明確＋本音まで踏み込む** | 「誰のどんな悩みを解決し、どんな理想の未来を叶えるか」を一言で言語化できるか（売れるnote3条件①）。訴求はメリット止まりじゃなく**ターゲットの本音の状態変化**まで（「連絡が返ってくる」でなく「自発的にLINEしてきてほしい」）。根拠は target_research.md のセクション番号 or Web検索の生の言葉で示す |
| 2 | **文脈＝プレイヤー感** | 普段の発信・ストーリーの流れと繋がってるか（売れるnote3条件③）。演者が「実際に自分がやった」の一人称で語れる手法か（自爆エピソード・落差ストーリーと接続）。**体験の裏付けが立たないコンセプトはフェイク臭で却下**（リアリティが必須＝理論②） |
| 3 | **独自の切り口** | ターゲットの常識を洗い出し、**完全否定型／問題すり替え型／逆転思考型**のどれかで切り込めてるか（理論②）。**手法に固有の名前を付ける**（競合レター参照：「ズルい復縁の法則」等）。よくある方法（冷却期間・自分磨き・LINEテク一般論）と同じに見えたら再設計。※「独自」は**市場の常識・競合に対する話**。自社アカ同士の被りは気にしない（売れた切り口を複数アカで使い回すのはむしろ正解）。被りを避けて売れないコンセプトになるのが一番のゴミ（阪本判断2026-07-03） |
| 4 | **note連動**（問題教育の前提条件） | このコンセプト＝ポストで認識させる問題点を解決する手段になってるか（`思考スキル_問題教育設計.md`）。問題点が立たないコンセプトはバズっても売れない |
| 5 | **圧倒的自信＋単価設計** | 「買ってくれたら人生変わる」と言える中身にできるか（売れるnote3条件②）。単価は初動980円→口コミ10〜20部→値上げの階段か、アカ内構成（メイン2,980＋サブ1,980×2＋ミニ980＝月20-30万の型）のどこに置くか言えるか（てっど8項目①③） |

### 注意事項

- AIキットの商品名（例：「ズルい復縁術」）の**丸パクリは禁止**。採用する場合も自分なりの言い換えで再構成
- **売れた切り口の再利用**：`商品コンセプト台帳.md` を読んで、**他アカで売れてる切り口があればそれを最優先で流用する**（実証済み＞新案。名前と演者の色だけ変える）

## Step 5：固定CTA決定（5案出し）

### 要素分解

CTAに必要な5要素：
- 数字×短期間（「1ヶ月で」「7日で」）
- 簡単性（「勝手に」「自分から」「何もしないで」）
- 結果直接（「戻ってくる」「言わせる」）
- 商品ジャンル名（「○○術」「○○メソッド」）
- ターゲット属性（「重い女」「回避型」など）

### 思考フロー

1. 上記5要素を変えながら5案生成
2. AIキット商品名の**丸パクリ回避**を確認
3. 5案提示 → 議論 → 1案確定

### 既存実例

- ハクオウ：「心理学を使って7日間で簡単に復縁できる復縁心理術」
- あや：「重い女のまま1ヶ月で回避型彼氏が戻ってくる逆転復縁術」

---

## Step 6：プロフ肩書き決定（5案出し）

### 参考構造

CSVの実物プロフ：
- もも：「回避型彼氏に振られたメンヘラが復縁成功／復縁する方法や関係修復のコツ」
- 復縁みるく：「メンヘラが原因で誕生日に振られたけど、そこから1ヶ月で復縁／現在復縁して2年目」
- シン：「1回目は喧嘩別れ、2回目は『もう連絡して欲しくない』と突き放し既読無視を半年間し...」

### 思考フロー

1. 落差ストーリー型／自爆エピソード列挙型／期間×結果型 の構造から派生
2. 5案提示 → 議論 → 1案確定
3. **1〜2行で読み切れる長さ**（45〜60字目安）

### 既存実例

- あや：「メンヘラで完全ブロックされたけど、3週間で立ち直って回避型彼氏と1ヶ月で復縁／今は溺愛される側で2年目」

## Step 7：ストーリー強度決定（Web検索必須）

**ここがアカウントのリアリティの核。省略不可。**

### 必須Web検索

以下のキーワードで検索する：

```
- 知恵袋 復縁 メンヘラ 追撃LINE 体験談
- ガルちゃん 振られた直後 やってしまったこと
- ガルちゃん 元彼を吹っ切れたきっかけ
- note 彼に振られたメンヘラが復縁できた理由
- 復縁 失敗談 LINE 既読スルー ブロック
```

ジャンルが回避型復縁以外なら、テーマに合わせて検索ワードを変える：
- 浮気テーマ：「知恵袋 浮気された 復縁 体験」「ガルちゃん 浮気彼 戻った」
- 遠距離：「知恵袋 遠距離 別れ 復縁」
- 不安型：「知恵袋 メンヘラ 振られる 体験」

### 抽出する素材

実体験から以下を抽出：

| カテゴリ | 例（あや作成時の抽出） |
|---|---|
| 自爆行動（リアル） | 鬼LINE毎日連投／週3回以上の電話／「他の女と話さないで」束縛／サブ垢24時間監視／家まで様子見に行く |
| 復縁後の彼の本音セリフ | 「雰囲気変わったね」「最近SNS見てて変わったって思ってた」「連絡きて嬉しかった」 |
| 数字データ | 3日で5キロ痩せる／月3万占い／3週間立ち直り期間 |
| 別れの本当の理由 | 「俺じゃ幸せにできない」「お前のこと好きすぎて応えられない」 |
| ブロック理由 | 「もう関わりたくない」「お前のメッセージ見て自分が壊れそうだった」 |

### 反映先

抽出した素材を **アカウント設定_〇〇.md** の以下に反映：
- ストーリー（どん底→転機→現在）
- リアリティ素材（自爆エピソード／彼の本音セリフ／数字データ）

### NG

- 「だいたい知ってる」で検索を省略する
- AIの想像で素材を埋める（リアリティが消える）
- ハクオウ過去ポストを参考にする（品質低い）

## Step 9：引き継ぎ書を生成（合体だけ・整形しない）

外注に渡すための引き継ぎファイルを、Step 1〜7で生成した素材を**そのまま合体**して作る。

**出力先**：`{アカウント名}/引き継ぎ_{アカウント名}.md`

### 合体する素材（順番にconcat）

```
## 1. アカウント設定（Step 3完成版）
→ アカウント設定_〇〇.md の中身をそのまま貼る

## 2. 商品コンセプト議論ログ（Step 4）
→ 出した5案 + 採用1案 + 不採用4案の理由メモをそのまま貼る

## 3. 固定CTA議論ログ（Step 5）
→ 出した5案 + 採用1案 + 不採用4案の理由メモをそのまま貼る

## 4. プロフ肩書き議論ログ（Step 6）
→ 出した5案 + 採用1案 + 不採用4案の理由メモをそのまま貼る

## 5. Web検索リサーチ生素材（Step 7）
→ 知恵袋／ガルちゃん／note等で集めた生の引用・抽出メモをそのまま貼る
   - 自爆エピソード（鬼LINE、週3電話、サブ垢監視 等）
   - 復縁後の彼セリフ（「お前のこと諦めきれなかった」等）
   - 数字データ（3週間／1ヶ月／月3万占い 等）
   - 別れの本当の理由・ブロック理由

## 6. （任意）外注向け作業依頼テンプレ
→ 必要になった段階で阪本さん本人が追記。最初は空でOK
```

### 絶対ルール

- **要約しない**：5案議論ログは「結論だけ」じゃなく出した5案全部含める
- **整形しない**：見出しだけ揃えて、中身は生のまま
- **サマリー化しない**：「このアカウントの本質は〜」みたいなまとめ文は不要

これは外注が**素材プールから自分で組み立てる**ためのファイル。整形すると素材が減って外注の手札が狭くなる。

## このスキルの本質（最重要）

**「テンプレを渡すんじゃなく、素材を読み込ませて毎回ゼロベースで思考させる」**

ナレッジ読み込み（Step 0）と Web検索（Step 7）は**絶対に省略しない**。

ショートカットしようとした瞬間、リアリティが消えてAI感が出る。
このチャットで「あや」を作った時のような議論プロセス（5案出し→ディスカッション→1案確定）を毎回再現することで、受講生個別のキャラが立ち上がる。
