# 判断軸：恋愛系ポスト作成 v4・圧縮版

**v3との違い**：抽象結晶化Phase記述／検証可能な予測テスト等を削除。実運用で必要な「判定アルゴリズム」と「キャラ適性フィルタ」だけ残す。

---

## 売れるポストの5要素連動構造

| 要素 | 中身 | ポストでの実装場所 |
|---|---|---|
| ① 読者の現在地 | 今の行動／信念／感情 | 段落1-2 |
| ② 読者の目的地 | 理想の未来（ベネフィット） | 段落7-8 |
| ③ ギャップの原因 | 問題点（信じてる常識） | 段落3 |
| ④ 橋渡し役 | キャラ（経験者／専門家／男性当事者） | キャラ設定全体 |
| ⑤ 橋渡しの手段 | ストーリー／セリフ／実例 | 段落4-5 |

**売れない＝5要素のどれかが欠けてる**。

---

## レバレッジ優先順位

| 優先度 | 要素 | 理由 |
|---|---|---|
| 1 | **③ギャップの原因** | noteとの連動の核。最大レバレッジ |
| 2 | ⑤橋渡しの手段（段落4） | 説得力の源。リアリティ素材で底上げ |
| 3 | ①現在地 | 抽象→具体化で自分事化が爆増 |
| 4 | ②目的地 | ベネフィットを本音レベルに |
| 5 | ④キャラ | 設定固定なので毎回最適化できない |

**STEP最重要：③ギャップの原因（問題点定義）**。ここがズレると他を磨いても売れない。

## フック判定アルゴリズム（IF-THEN）

ユーザーから受け取ったフックの**開口部キーワード**で自動判定：

| 開口部キーワード | カテゴリ | 参照先 |
|---|---|---|
| 「経験談ですが」「私の経験上」「復縁して分かったんだけど」「〜と復縁して〜」 | A：経験者の暴露系 | パターン集A |
| 「ちょっと本音を言うと」「男目線でいうと」「男の本音を言うと」「断言するけど」「これあんまり言いたくないんだけど」 | B：男性当事者の本音系 | パターン集B |
| 「復縁した彼が言ってたけど」「復縁した彼に〜って聞いた」 | C：セリフ引用系 | パターン集C |
| 「『〇〇』と思うと〇〇」「結局、〇〇のは〇〇」「〇〇は『〇〇』」 | D：衝撃的命題系 | パターン集D |
| 「諦めなくていい」「絶対戻れる」「挽回できる」「辛いことだよね」 | E：励まし型 | パターン集E |
| 「実は、〇〇」「意外かもだけど」「〇〇は嘘」「〇〇より圧倒的に〇〇」「これ女性は恐らく知りませんが」「〇〇は〇〇じゃない」「〇〇するんじゃなくて〇〇」 | F：常識破壊系 | パターン集F |
| 「男は〇〇とは思っていない」「男は〇〇できない」「男って〇〇でも〇〇」「男性って〇〇」「〇〇な男ほど〇〇」 | G：男の心理断言系 | パターン集G |
| 「今でこそ〇〇だけど、〇〇」「〜すぐの頃は〇〇だったけど、〇〇」 | H：Before/After落差系 | パターン集H |

マッチしない場合：
- 起点が「読者の行動描写」→ F-6 or B-1
- 起点が「読者の信念引用」→ F-3 or F-7
- 起点が「読者の悩み」→ E-4 or 別カテゴリ

## キャラ適性フィルタ

カテゴリ判定後、受講生のキャラ設定と照合：

| キャラタイプ | 使えるカテゴリ | 使えないカテゴリ |
|---|---|---|
| **経験者女性**（みるく／もも／R等） | A／E／F／G／H／D-1／D-2 | C-1（=自分が復縁した彼に聞いた、なら可）／B（男性視点） |
| **男性当事者**（凪／はる／ななせまる／シン） | B（全部）／A-3／A-4／A-5／F／G／D | E（励ましは女性キャラ向き）／H-1（経験者女性のみ） |
| **専門家**（ハクオウ等） | F／G／D／A-2／A-3（「相談者で〜」と変形）／一部C | B（男性当事者向き）／H（経験者向き）／E（励まし系は弱い） |

キャラと不適合なフックが来たら：「このキャラだとこのフックは使えない。代替案として〇〇カテゴリに変換しますか？」と提示。

## 構造選択（フック型に応じた本文4パターン）

| フック型 | 本文4パターン | 理由 |
|---|---|---|
| F系（常識破壊）／G系（男の心理断言） | A：相談者成功型 | 反転の証拠を相談者story で示す |
| C系（セリフ引用）／A-3〜A-5（経験者の暴露） | C：男の本音引用型 | セリフを段落の主役に |
| D系（命題）／F-1（実は型） | D：メカニズム解説型 | 原理を中心に置く |
| E系（励まし）／F-3（嘘） | A：相談者成功型 ＋ B：失敗警告型 のミックス | 希望と緊張感のバランス |
| H系（落差） | A：相談者成功型 | 落差そのものが事例 |

※サブパターン3 or 4は保留中（テスト後決定）。

## 5要素チェック（出力前）

```
□ ①読者の現在地が具体的（「3週間既読スルー」「毎日LINE送ってブロック」等）
□ ②目的地（ベネフィット）が本音レベルで明示
□ ③ギャップの原因（問題点）がnoteの解決領域と一致
□ ④キャラの一貫性（語尾／一人称／フレーズ）
□ ⑤橋渡しの手段が「具体的体験」になってる（一般論じゃない）
```

5要素連動が崩れたら、レバレッジ優先順位の高い方（③→⑤→①→②→④）から修正。

## 境界条件（このスキルが機能しない場面）

| 場面 | 対応 |
|---|---|
| 短期バズ狙い（売上目的じゃない） | 別フレーム使用 |
| ブランディングポスト | 別フレーム使用 |
| 既存の濃いファン向け | 5要素揃え過剰、簡略可 |
| 超低単価商品（500円以下） | 衝動買いで売れる、簡略可 |
| 時事ネタ便乗バズ | 連動より話題性優先で別ロジック |

## まとめ

判断軸の核：
1. **5要素連動構造**で本質をブレない
2. **フックキーワードで自動判定**して認知コスト下げる
3. **キャラ適性フィルタ**で誤用回避
4. **レバレッジ優先順位**で修正の効率化

詳細記述（抽象結晶化Phase / 検証予測テスト）は実運用で使ってないので削除。
