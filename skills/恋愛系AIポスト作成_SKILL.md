# 恋愛系AIポスト作成スキル

恋愛系演者運用の**全工程の統括スキル**。
note立ち上げ（記事作成）と Xポスト量産（短文SNS）の両方をカバーする。

起動したら**ユーザー確認は最小限、ノンストップで最後まで走る**。

---

## 起動方法

### A. 全工程ノンストップ（新演者立ち上げ）

- 「{演者名}を立ち上げて」
- 「新演者作って」
- 「恋愛系の新規アカウント立ち上げて」

→ Phase 1（コンセプト設計）から Phase 9（Xポスト量産）まで一気通貫。

### B. 単体フェーズ起動（既存演者の部分作業）

- 「{演者名}のポスト{N}本作って」 → **Phase 9 だけ**（Xポスト量産）
- 「{演者名}のletter書いて」 → **Phase 2 だけ**
- 「{演者名}のcontent書いて」 → **Phase 3 だけ**
- 「{演者名}の画像生成」 → **Phase 4 だけ**
- 「{演者名}のnote投稿スクリプト走らせて」 → **Phase 5〜6**
- 「{演者名}のサムネ文字案」 → **Phase 7 だけ**
- 「{演者名}の引き継ぎ書だけ作って」 → **Phase 1のStep 9 だけ**
- **「{演者名}を改善ループに登録して」** → **Phase 10 だけ**（hakuoo-analytics の3日サイクル自動投稿に乗せる）
- **「{演者名}を3日サイクルに乗せて」** → **Phase 10 だけ**（上と同じ）

### C. 範囲指定起動（複数フェーズ連続）

- 「{演者名} フェーズ2から」 → letter以降全部
- 「{演者名} フェーズ4まで」 → 画像生成までで止める
- 「{演者名} フェーズ2〜4」 → letter→content→画像 だけ

## 設計思想（ユーザー確認は最小限）

過去の断続原因を全部このスキルで潰す：

| 過去の断続ポイント | このスキルでの対処 |
|---|---|
| letter保存忘れ | 各Phaseで Write必須、チャット出力のみで完了禁止 |
| 画像文字化け（Nano Banana 2） | **`gemini-3-pro-image-preview`（Nano Banana Pro）固定** |
| デザイン5択で毎回ユーザー選択待ち | 初回のみ提示、選んだら **アカウント設定に design_id 保存**、2回目以降は無確認 |
| 結合MD・スクリプト生成の手作業 | Phase 5で自動展開 |
| 監視員チェックで毎回ユーザー確認 | B以下を自動書き直し、ユーザー確認なし |
| サムネ商品名の改変 | **メイン=商品名そのまま固定**、3案バリエはラベル＋サブのみ |

ユーザー確認を入れるのは：
- フェーズ1の5案議論（コンセプト/CTA/プロフ）
- フェーズ4のデザインID初回選択（5択、登録後2回目以降は無確認）
- 最後の公開作業（noteブラウザでの有料エリア境界・価格設定・公開ボタン）

それ以外は止まらない。

## フェーズ構造

### Phase 1: コンセプト設計＋引き継ぎ書合体（阪本さん本人推奨）

**参照**: `スキル_恋愛コンセプト設計.md`

- アカウント設定_〇〇.md を作成（Step 1〜7）
- Step 8 の出力前チェックを通す
- Step 9 で引き継ぎ_〇〇.md を合体生成（生のまま、整形しない）

**出力（Write必須）**：
- `{アカウント名}/アカウント設定_{アカウント名}.md`
- `{アカウント名}/引き継ぎ_{アカウント名}.md`

阪本さんが自分でやる場合はこのフェーズだけ完走させて、フェーズ2以降は外注に渡してもOK。

### Phase 2: letter作成

**参照**: `スキル_恋愛レター作成.md`

#### Step 2-0：競合レター4本の構造分析（必須・冒頭で実施）

`競合レター集/恋愛復縁/*.md` の4本（凪×2、もも×2）を**Phase 2 の最初に読み込む**。
6フェーズ構造／仮想敵／数字権威／本音セリフ密度／当事者ストーリーの出し方を横断抽出してから、本演者の素材で再構築する。

過去のあや立ち上げ時に context-crystallizer で行った分析と同じタイミング・同じ素材で実施することで、同等クオリティを再現する。

#### Step 2-1：letter本体生成

- 6フェーズ構造（A防衛解除→B常識破壊→C責任解除→D証拠三層→E未来3段階→F二択）
- 恋愛特化6発露パターン（頑張りたくない開口部／本音セリフ5+／数字権威／当事者性／相性論否定／テクニック語回避）
- キャラ別語尾・絵文字をアカウント設定から読み込んで適用
- 必須マーカー導入文3個（序盤B型／中盤D型／後半A型）を本文に埋め込む
- 監視員チェック → B以下を自動書き直し → 完成版

**★AI感監視員パス（必須・letter完成後）**: `.claude/skills/AI感監視員/SKILL.md`
- 上の品質監視員（A/B/C評価）とは別物。AI感・業者感の**単語/言い回しレベル**を6軸で通す
- 構成・主張は変えない。引っかかった単語・語尾・記号・脚色だけ書き換える
- **書き換えまで実行し、書き換え後の完成版を阪本さんに提示する**（指摘だけで終わらせない）
- 演者プロファイル（語尾・絵文字・温度）を読んでから当てる

**出力（Write必須）**: `{アカウント名}/letter.md`（AI感監視員の書き換え後の版を保存）

**重要**: チャット出力だけで完了させない。必ず Write でファイル保存する。これが過去の断続原因No.1。

#### Step 2-2：実践者の声カード（必須・letterの一番最初に差し込み）

**目的**: セールスレターの**冒頭**に「いただいた声」3枚を置き、読者が掴みたい未来を先に見せる。

手順：
1. **letter本体から「ターゲットの悩み → 掴みたい未来」を抽出**（例：復縁＝「彼の方から来てくれた」）。声はこの未来が叶った形で書く
2. **声を3つ生成（中くらい1つ＋長いの2つ）**。全部別人・別エピソード・別年代。`まさか彼の方から来てくれるなんて` のような実体験トーン
   - **★声の文章もAI感監視員（`.claude/skills/AI感監視員/SKILL.md`）を通す。** 証言がAIっぽいと逆に嘘くさくなる。語尾・長さをばらし、テンプレ臭・業者感を消す
   - **★3人を「明らかに別人」に書き分ける**（パッと見で全部同じ人＝やらせ、を回避）：
     - 文体を変える：1人は絵文字を使うタイプ（`🥺😭💦😳` 等を自然に）、1人は淡々と句点で区切る、1人は普通。同じ語尾・同じ言い回しを3人で使わない
     - 年代をばらす（20代／30代前半／30代後半 等）
     - `face` ヒントを明確に変える：髪型・雰囲気・メガネ有無などで別人とわかるように（例 `20代前半 ふんわり茶髪`／`30代後半 黒髪ロング メガネ`／`30代前半 ショートヘア`）。※カードのアイコンのリング色は1枚ごとに自動で変わる
3. 声を `{アカウント名}/_声.json` に書く（形式 `[{"text":"...","attr":"20代女性","face":"20代前半 明るい雰囲気 ふんわり茶髪"}, ...]`）
4. **カード生成**：`python note_tools/実践者の声_カード生成.py --voices {アカウント名}/_声.json --output {アカウント名}_note_images/声カード/ --seed {アカウント名}`
   - 文字はPIL合成・左寄せ＋禁則／**文ごとに改行＋文間に余白＋縦中央寄せ**で読みやすく（スクリプトが自動。1ブロックでびっしり詰めない）／顔だけGemini生成＋モザイク（本人っぽいが特定不可）／素朴・可愛い・派手すぎない（派手＝嘘っぽい）
   - 本文に絵文字が混ざってもカラー表示される（Segoe UI Emoji対応済み）／アイコンのリング色は1枚ごとに自動で変わる（別人化）
   - **★デザインは毎回変える（固定テンプレ厳禁）。** 全演者・全noteで同じ見た目だと「これ見たことある＝やらせ」とバレる
     - `--seed {アカウント名}` で演者ごとに配色・見出し・装飾が自動で変わる。または `--style` を明示（`rose_band/sage_sprig/lavender_soft/cream_sprig/blue_dots/peach_bloom`）して毎回ずらす
     - さらにアドリブで変化させてよい：花のあしらい・色味・ラベル文言（「いただいた声」以外に「届いた声」等）・属性（年代）・顔の人選。**「一例」であって正解の型ではない**
   - 出力先は他の画像と揃える：`{アカウント名}_note_images/声カード/声カード_01〜03.png`
5. **letterの一番最初（H1タイトル直後・「忠告」より上）に、声を導くリード行を1本だけ置く**（これが画像のアンカー）
   - 例（演者の口調で・AI感監視員通過）：`先に、実際にいただいた声をいくつか紹介させてね🌷`
   - ※ letter.md には画像を直書きしない。画像挿入は Phase 6 の note貼り付け（アンカー方式）で自動で行う

**画像挿入は手貼りしない。** 既存画像と同じ「アンカーテキスト方式」（`IMAGE_MAP`）で Phase 6 に自動挿入させる（→ Phase 5 参照）。

**出力**: `{アカウント名}_note_images/声カード/声カード_01〜03.png` ＋ letter冒頭のリード行

### Phase 3: content作成（有料パート）

**参照**: `スキル_恋愛コンテンツ本編作成.md` ／ `AIキット_コンテンツ作成ルール.md`

- 6章構成、17,000〜20,000字
- 「はじめに → 章1〜6 → おわりに」
- マーカー導入文10個を本文中に自然な単独行で配置（40字以内、記事内ユニーク）
- キャラ口調は章全体で一貫（敬語と口語の混在NG）
- 4ブロック分割で書く（前半→中盤→後半→まとめ、ブロックごとに Write でファイル追記）
- 監視員チェック完了→B以下を自動書き直し

**出力（Write必須）**: `{アカウント名}/content.md`

### Phase 3.5: 図解デザイン選定

**参照**: `AIキット_図解デザイン監視員ルール.md`

1. アカウント設定に `design_id` 記載あり → 即進む
2. 記載なし → 5択提示してユーザー選択 → 選択したIDをアカウント設定に追記保存
3. 監視員5観点を自動チェック（B以下3つ未満なら通過、3つ以上なら別IDで再選定、最大2回）

5択（恋愛系）：
1. ソフトピンク × ホワイト
2. ダスティローズ × ダークネイビー
3. ラベンダー × アイボリー
4. コーラル × チャコール
5. カスタム

### Phase 4: 画像13枚生成（letter 3 + content 10）

**参照**: `AIキット_図解生成ルール.md`

- スクリプトを `note_tools/generate_{アカウント名}_images.py` に自動生成
- **MODEL_PRIMARY = "gemini-3-pro-image-preview"（Nano Banana Pro）固定**
- 命名: `{アカウント名 大文字}_L_01〜03 / {アカウント名 大文字}_C_01〜10`
- 1枚目（典型的に C_06：文字量多め）テスト→Read で文字化け確認→残り12枚

**出力**:
```
{アカウント名}_note_images/{ID}_*.png × 13枚
```

### Phase 5: 結合MD + note貼り付けスクリプト生成

**参照**: `AIキット_note貼り付けルール.md`

- letter.md + content.md を結合 → `note_tools/【note投稿用】{アカウント名}_完成版.md`
- letter冒頭の `# h1見出し` は1個だけ残す（noteタイトル用）、それ以外は `##` に降格
- `note_tools/AIキット_note自動貼り付け.py` をテンプレに `note_tools/{アカウント名}_note自動貼り付け.py` を自動生成
- IMAGE_MAP に13枚分を自動展開
- **★実践者の声カード3枚を IMAGE_MAP の先頭に登録**（Step 2-2で生成済み）：
  - `anchor` は letter冒頭のリード行（例：`先に、実際にいただいた声をいくつか紹介させてね`）。これで「いただいた声」が**本文の一番最初**に挿入される
  - **登録順は逆（声カード_03 → 02 → 01）**。同じアンカー行の直後に順に挿入すると逆順に積まれるため、逆登録で表示順が 01→02→03 になる
  - `image` は `{アカウント名}_note_images/声カード/声カード_0N.png`
- NOTE_ID/PASSWORD は既存スクリプト（`firm_oriole6694` / `shouta0123`）から流用

### Phase 6: 自動投稿実行

```bash
python note_tools/{アカウント名}_note自動貼り付け.py
```

- run_in_background: true で起動
- Monitor で `STEP|OK:|→|✓ 成功|✗|⚠|ERROR|画像挿入結果|全工程完了` をストリーム
- 13/13 完了を確認したら次へ

### Phase 9: Xポスト量産（Threads/X短文ポスト）

**参照**: `スキル_恋愛ポスト作成AI.md`（v4・最重要・このスキルが今もメイン武器）

このスキル（恋愛系AIポスト作成）の**本丸**。
note立ち上げが終わった後の日次運用、または既存演者のポスト量産依頼に対応する。

#### ★Phase 9 最上位フロー（v4.2・絶対遵守）★

**ポスト1本でも作成する前に、必ずこの順番で進める**：

1. **`{演者名}_セールスレター.md` または `{演者名}/letter.md` `{演者名}/content.md` を必ず読む**
   - 例：`ハクオウ_セールスレター.md` / `あや/letter.md`
   - 読まずにポスト作成は禁止

2. **セールスレターから 4要素を抽出**（思考スキル_本文作成.md の v4.2最上位フロー ステップ2 参照）：
   - 読者の悩み
   - **読者がやっちゃってる NG行動**（これがメインフックの素材になる）
   - 商品の提供価値（章タイトル・特典・1日目の内容等）
   - 商品の正解手順

3. **1ポスト = 1「やっちゃってる」指摘** がメインフックの基本構造
   - 「これ送ってる人」「これやってる人」など、読者の具体行動を1つピンポイントで指摘
   - 抽象構造の解説・専門用語は禁止
   - 「臨床心理学的に」「メカニズム」「ロック」「フェーズ」「印象固定」等は明確にNG

4. **ぶら下げは演者本人の失敗エピソード起点**
   - 「僕も振られた直後、〇〇してた」「あの時の僕は〇〇」
   - 数字や構造の解説より、場面・セリフ・気持ちを優先

5. **CTAは「商品の中身が見える言葉」で締める**
   - 「最初の1通の置き方」「1日目に何をするか」「彼が気にする側になる動き方」など
   - 「心理学ベースの〇〇プログラム」みたいな抽象ラベルだけのCTAは禁止

6. **構成比率：1日10本なら ロング6本＋短文4本**
   - ロング：上記のフルフローで作成
   - 短文：思考スキル_本文作成.md「短文ポスト 5型」から型①〜⑤を使い分け、120-140字、CTA無し

7. **★AI感監視員パス（必須・ポスト完成後）**: `.claude/skills/AI感監視員/SKILL.md`
   - 全ポスト（ロング＋ぶら下げ＋短文）を6軸で通す。特に軸3「業者感」（`Day○○`/`①②③`/`👇`一辺倒/テンプレCTA）と軸2「bot感」（語尾連発/絵文字単調）を重点チェック
   - 構成・主張は変えない。単語・語尾・記号・脚色だけ書き換える
   - **書き換えまで実行し、書き換え後の完成版（全ポスト）を阪本さんに提示する**（指摘だけで終わらせない）
   - 演者プロファイル（語尾・絵文字・温度）を読んでから当てる

#### v4.1 → v4.2 で削除した過剰ルール

以下のルールは「構造遵守ゲーム」を生んで専門用語・抽象解説を促進したため削除：
- 「L4 時間制限/書き換え/手遅れ/数字リスク テンプレ」（→ 場面で描く）
- 「CTA 型A/B/C テンプレサンプル」（→ 中身が見える言葉で）
- 「6層全項目チェック厳格運用」（→ 5層を「考える型」として使う）

これらが残ってる古いポストは、v4.2 最上位フローで書き直す対象。

ループ起点でフックパターン集だけ参照して書くのは禁止。

#### 単体起動の典型ケース

- 「{演者名}のポスト10本作って」 → 1日分のセット
- 「{演者名}のポスト30本作って」 → 3日分のセット
- 「{演者名}のポスト100本作って」 → 10日分のセット
- 「{演者名}のポスト300本作って」 → 1ヶ月分のセット（10本×30日）
- 「{演者名}のセールスレターから売れる切り口抽出して、ポスト10本」

#### 必須インプット

1. **演者名**（あや／うみこ／みき／…）
2. **演者プロファイル**（v4登録演者ファイル4セット：演者プロファイル／ハック／型／チェックリスト）が `SNSポスト作成キット_汎用版_展開/` 配下に揃ってるか
3. **素材源**：演者の **letter.md** または **セールスレター本文** または **content.md**
   - これらから「売れる切り口」を抽出してポストに反映する
4. **本数**（10／30／100／300）
5. **目的**（リスト獲得／note直販誘導／フォロワー増 等）

#### 実行ルール

- v4スキル（`スキル_恋愛ポスト作成AI.md`）の手順をそのまま回す（圧縮・要約しない）
- 各演者の既存登録ファイル4セットを読み込む（CLAUDE.md「マルチ演者対応」参照）
- セールスレター素材から「フック／本音セリフ／落差ストーリー／常識破壊」を抜き出してポスト構造に流し込む
- カテゴリ配分：A系（経験者暴露）／D系（命題）／E系（励まし）／F系（常識破壊）／H系（落差ストーリー）等
- フック集合格ラインチェックを各ポストごとに実行（メモリ：feedback_hook_checklist.md）
- 出力先：`generated_x_posts/{演者名}_post_batch_{YYYYMMDD}.json`

#### 大量本数（100〜300本）の分割実行

- 一度にやるとコンテキスト溢れて品質落ちる
- 10本×N回、または30本×N回でバッチ実行
- 各バッチで切り口・カテゴリ配分を変えて重複防止
- 最後にすべて結合した1ファイルにまとめる

#### 完了報告

```

- 切り口の元ネタ：{セールスレター名 or letter.md}
- 重複チェック：完了
```

## エラー時のリトライ・スキップルール

### 画像生成エラー
- 同モデルで3回リトライ → ダメならフォールバックモデル（gemini-2.5-flash-image）で3回 → それでも失敗ならスキップしてログ記録
- 13枚中12枚成功なら続行（足りない1枚は阪本さん手動補完）

### note貼り付けエラー
- アンカー未検出 → スクロール再試行 → 次の画像へ（後で阪本さん手動挿入）
- ログイン失敗 → スクリプト即停止、阪本さんに通知

### letter/content 監視員チェック失敗
- B以下が3つ以上 → 該当部分だけ書き直し（最大2回、3回目はAS均等で強制承認）
- ユーザー確認不要

## このスキルの本質

**「恋愛系演者運用の全工程を1スキルで統括する」**

カバー範囲：
- **note立ち上げ系**（Phase 1〜8）：コンセプト→letter→content→画像→note公開準備→サムネ文字案
- **Xポスト量産系**（Phase 9）：1日10本〜1ヶ月300本まで、セールスレターから売れる切り口を抽出して短文ポスト化

阪本さんが「あや立ち上げて」と言えば全工程ノンストップ、「うみこのポスト100本作って」と言えばPhase 9だけ起動。

新演者立ち上げ時も日次運用も、起動文1つに圧縮するのがこのスキルの目的。

### 既存スキルの圧縮はしていない

このスキル本体は**統括役（指揮者）**で、各フェーズで対応するスキル本体（スキル_恋愛コンセプト設計.md／スキル_恋愛レター作成.md／スキル_恋愛コンテンツ本編作成.md／**スキル_恋愛ポスト作成AI.md（v4）**／AIキット_各種.md）を**そのまま参照する**。中身を取り込んだり要約したりしていない。
