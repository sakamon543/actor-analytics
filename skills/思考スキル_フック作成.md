## 設計原則（最低限）

### 1.5秒の決断ルール
人は1.5秒以内に「読むか／スクロールするか」を決める。フック1行目の威力が全て。

### 具体性 > 曖昧
「shocking」「secret」等の煽り単語はクリックベイト判定で逆効果。
**具体名詞・具体数字・具体セリフ**が刺さる。

### 1行目力チェック（出力前必須）

```
□ 文字数：パターン集v4実測表のサブspec範囲内
□ Setup + Promise の構造（ギャップ + 答えの予感）
□ 命題型（断言）になってる
□ 「〇〇な子に伝えたい」等の弱い前置きなし
□ 抽象語（〇〇的、〇〇性、〇〇化）なし
□ 煽り語（shocking、すごい、ヤバい）なし
□ 具体性（数字、セリフ、固有名詞）が入ってる
□ 1.5秒で「え？」と思える
```

## 3ステップ思考フロー

### STEP 1：テーマからサブパターンを選ぶ

1. noteの解決領域から、今回のテーマを1つに絞る
   - NG：「男の心理を伝えたい」（広すぎ）
   - OK：「既読スルーは脈なしじゃないと伝えたい」（具体的）
2. パターン集v4から、テーマに合うサブパターン（A〜H、65個）を1つ選ぶ
3. 直近5本と被らないものを優先

### STEP 2：変数を埋める（3段階フロー）

**必ずこの順序で素材を選ぶ。AIの想像で埋めるのは禁止**。

```
[第一選択] 素材実例集 Part 1 のCSV直接実例
          → 該当サブパターンに直接実例があれば、それを最優先で転記
          
[第二選択] 素材実例集 Part 2 の観察ワード集
          → 男の本音セリフ／心理メカニズム／落差描写から具体ワードを引く

[第三選択] 4テスト合格を条件に新規生成
          □ イメージテスト：読んで具体シーン／動作が浮かぶか
          □ 圧縮テスト：1〜2語で観察を圧縮してるか
          □ 造語感テスト：「あ、それ」と新鮮さがあるか（AI造語にならない範囲）
          □ 観察由来テスト：「なんで思い付いた？」に「実生活で見た」と答えられるか
          
          4テスト全部通らない → 変数を作り直す or 別テンプレに切り替える
```

### STEP 3：機械チェック（出力前必須）

```
□ 文字数：パターン集v4の該当サブspec範囲内（実測表で確認）
□ 開口部固定句：1字単位で照合
□ 末尾固定句：1字単位で照合
□ 句読点：「、」が指定位置のみ／余計な「、」追加なし
□ メインポスト全体：100-150字（hook + 続き2-4文）
```

1つでも違反 → そのhookを破棄して再生成。spec違反のまま出力するのは禁止。
