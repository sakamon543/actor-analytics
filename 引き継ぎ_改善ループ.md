# 改善ループ引き継ぎガイド

既存の恋愛系アカウントを自動改善ループに乗せるための手順書。

---

## この仕組みが何をするか

3日に1回（月木日 09:00 JST）、以下が全自動で動く：

```
① スレッズ自動運用マシーンから直近の投稿データを取得
② 伸びた/伸びなかったポストを分析
③ Claude APIで次の3日分30本を生成（evaluation_criteria の5問チェック通す）
④ スレッズ自動運用マシーンに予約投稿
```

これが全演者分、自動で回り続ける。

---

## 必要な情報（1アカウントあたり）

ユーザーが以下の3つを渡す。AIがこれをもとに `アカウント情報総合管理.md` を作成し、改善ループに組み込む。

### 1. 演者名と商品情報（必須）
- 演者名（キャラ名）
- 商品名、価格
- 商品が解決する問題
- 固定CTA文言

### 2. キャラのコンセプト設計（必須）
- 肩書き、性別/年齢
- 一人称（僕/私/俺）、口調タイプ（専門家系/経験者女性/男性当事者）
- キャラ的フレーズ集（3〜5個）
- 語尾と文体ルール
- 使えるカテゴリ / 使えないカテゴリ / 絶対NG

既存のアカウント設定ファイル（`アカウント設定_〇〇.md`）や引き継ぎファイルがあれば、そこから抽出できる。

### 3. スレッズ自動運用マシーンのトークン（必須）
- `taf_xxxxx...` の形式
- スレッズ自動運用マシーンの管理画面で確認

### ターゲットリサーチについて（重要）

ターゲットのペルソナ情報は `knowledge/shared/target_research.md` に全演者共通で整備済み（400件以上の生データ、18セクション構成）。
**個別アカウントに浅いリサーチ情報を持たせない**。浅い情報を参照するとポスト生成・評価（STEP 4の5問チェック）の精度が落ちる。
ポスト生成時は必ず `knowledge/shared/target_research.md` を参照させる。

### 技術的な前提
- GitHubアカウント：https://github.com/sakamon543/actor-analytics にアクセスできること

---

## 手順

### ステップ1：アカウント情報総合管理.md を作成・配置する

AIが上記の4情報をもとに `accounts/演者名/アカウント情報総合管理.md` を作成する。

ファイルの構成（実例は `accounts/ハクオウ/アカウント情報総合管理.md` を参照）：

```
## メタ情報           ← AIが自動で埋める
## 改善ループ設定     ← AIが自動で埋める（token_env名、enabled、API情報）
## キャラ情報         ← 上記2から作成
## 商品情報           ← 上記1から作成
## ターゲット         ← 共通ペルソナ（knowledge/shared/target_research.md を参照）
## 売れたポスト       ← 運用後に蓄積（初期は空でOK）
## リアリティ素材     ← 運用後に蓄積（初期は空でOK）
## ポスト作成ルール   ← 上記2から作成
## 投稿スケジュール   ← デフォルト値で自動生成
```

**ターゲット情報は個別に持たない。** `knowledge/shared/target_research.md`（全演者共通）が唯一の正とする。

以下のセクションはAIが自動で生成する（ユーザーが手で書く必要はない）：

```
## メタ情報

| 項目 | 内容 |
|---|---|
| 演者名 | （演者名） |
| アカウントID | @xxxxx |
| プラットフォーム | Threads |
| 立ち上げ日 | YYYY-MM-DD |
| 最終更新 | YYYY-MM-DD |

## 改善ループ設定

| 項目 | 値 |
|---|---|
| token_env | （演者名）_TOKEN |
| enabled | true |
| 運用開始日 | YYYY-MM-DD |
| 運用終了日 | YYYY-MM-DD |
| サイクル | 3日ごと |
| 知識ファイル | knowledge/shared/（全演者共通） |
```

実例は `accounts/ハクオウ/アカウント情報総合管理.md` を参照。

---

### ステップ2：アカウント帳簿総合.yaml にアカウントを追加する

ファイル：`アカウント帳簿総合.yaml`（リポジトリ直下）

既存のブロックをコピペして、値を書き換える：

```yaml
  - name: 演者名
    token_env: 演者名_TOKEN
    master: accounts/演者名/master_100posts_ranked.json
    account_setting: accounts/演者名/アカウント情報総合管理.md
    start_date: 2026-XX-XX
    end_date: 2026-XX-XX
    enabled: true
    notes: メモ（自由記述）
```

**各フィールドの意味：**

| フィールド | 何を書くか |
|---|---|
| `name` | 演者名（フォルダ名と一致させる） |
| `token_env` | ステップ3でGitHubに登録するトークンの名前。`演者名_TOKEN` の形式 |
| `master` | 最初の投稿セット（なければ空でもOK、fetchで自動蓄積される） |
| `account_setting` | ステップ1で置いたファイルのパス |
| `start_date` | 改善ループの開始日 |
| `end_date` | 改善ループの終了日 |
| `enabled` | `true` = 自動改善ON / `false` = 一時停止 |
| `notes` | 自由メモ（スクリプトは読まない） |

---

### ステップ3：GitHubにトークンを登録する

1. 以下のURLを開く：
   https://github.com/sakamon543/actor-analytics/settings/secrets/actions/new

2. 入力する：
   - **Name**：ステップ2の `token_env` に書いた名前（例：`UMIKO_TOKEN`）
   - **Secret**：スレッズ自動運用マシーンのトークン（`taf_xxxxx...`）

3. 「Add secret」を押す

## 変更をGitHubに反映する

ステップ1〜2でファイルを追加・編集したら、GitHubにpushする。

Claude Codeで作業してる場合：
```
cd hakuoo-analytics
git add accounts/演者名/ アカウント帳簿総合.yaml
git commit -m "add: 演者名を改善ループに追加"
git push
```

## 動作確認（手動テスト）

初回はGitHub Actionsを手動で動かして確認する：

1. 以下のURLを開く：
   https://github.com/sakamon543/actor-analytics/actions/workflows/3day-cycle.yml

2. 右上の「Run workflow」→「Run workflow」を押す

3. 実行ログで該当演者の処理が通ってるか確認する

---

## 一時停止と再開

一時停止：`アカウント帳簿総合.yaml` の該当演者の `enabled` を `false` に変えてpush。
再開：`true` に戻してpush。

データやファイルは一切消えない。

---

## アカウントを止めずに改善ループだけ止めたい場合

同じく `enabled: false` にする。スレッズ自動運用マシーン側の投稿は別管理なので影響しない。改善ループが止まるだけ。

## トラブルシューティング

### GitHub Actionsが動かない
- https://github.com/sakamon543/actor-analytics/actions で直近の実行ログを確認
- 赤い×マークの実行をクリックすると、エラー内容が見える

### 特定の演者だけスキップされる
- `アカウント帳簿総合.yaml` の `enabled` が `true` になってるか確認
- GitHub Secretsに `token_env` と同じ名前でトークンが登録されてるか確認
- トークンが期限切れになってないか確認

### ANTHROPIC_API_KEY のエラー
- Claude APIのキーがGitHub Secretsに登録されてるか確認
- 登録先：https://github.com/sakamon543/actor-analytics/settings/secrets/actions
- 名前：`ANTHROPIC_API_KEY`
- 値：Anthropicのダッシュボード（https://console.anthropic.com/settings/keys）で取得

### 生成されたポストの品質が低い
- `knowledge/shared/evaluation_criteria.md` と `knowledge/shared/target_research.md` を更新する
- evaluation_criteriaの5問チェックが機能してない場合は、target_research.mdのセクション内容が古い可能性がある

## ファイル構成（全体像）

```
hakuoo-analytics/
├ アカウント帳簿総合.yaml                          ← 演者名簿（ここに追加するだけ）
├ accounts/
│  ├ ハクオウ/
│  │  └ アカウント情報総合管理.md         ← 演者の全情報
│  ├ うみこ/                             ← 演者ごとにフォルダ
│  │  └ アカウント情報総合管理.md
│  └ ...
├ analytics/
│  ├ ハクオウ/
│  │  ├ posts_db.json                    ← 投稿データ（自動蓄積）
│  │  └ next_batch.json                  ← 次の3日分（自動生成）
│  └ ...
├ knowledge/
│  └ shared/
│     ├ evaluation_criteria.md           ← フック評価基準（全演者共通）
│     └ target_research.md              ← ターゲットデータ（全演者共通）
├ note_tools/
│  ├ run_3day_cycle.py                   ← メインスクリプト
│  ├ fetch_threads_analytics.py          ← データ取得
│  ├ analyze_post_performance.py         ← 分析
│  ├ hook_improve.py                     ← Claude APIで生成
│  └ schedule_posts.py                   ← 予約投稿
└ .github/workflows/
   └ 3day-cycle.yml                      ← GitHub Actions設定（3日サイクル）
```
