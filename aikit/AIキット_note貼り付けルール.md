# AIキット note自動貼り付けルール

> **このルールの役割**
> コンテンツ作成ルールのPhase 2（図解生成）完了後、レター＋コンテンツの結合テキストと図解画像をnoteに自動投稿する。
> テキスト一括ペースト → アンカーテキスト方式で画像挿入 → 下書き保存 までを1スクリプトで完結させる。

---

## 0. 前提

- noteのエディタはProseMirror（リッチテキストエディタ）で動いている
- Markdownをコピペするとnote側がリッチテキストに自動変換するが、全ての記法が完璧に変換されるわけではない
- 投稿スクリプトは `note_tools/AIキット_note自動貼り付け.py` をベースにカスタマイズして使用する
- noteログイン情報はコンセプト設計ルール（■1 初期設定シート）で取得済みの前提

---

## 1. noteで使っていい記法 / 使ってはいけない記法

### ✅ 使っていい記法

| 記法 | Markdownの書き方 | noteでの表示 |
|------|----------------|-------------|
| 大見出し | `## テキスト` | 大見出し（太字＋大きい文字） |
| 小見出し | `### テキスト` | 小見出し（太字＋中くらいの文字） |
| 太字 | `**テキスト**` | 太字（条件あり、下記参照） |
| 引用 | `> テキスト` | グレー背景の引用ボックス |
| 空行 | 空の行 | 段落区切り |

### ❌ 使ってはいけない記法

| 記法 | 何が起きるか | 代替手段 |
|------|------------|---------|
| `- リスト` `* リスト` | バグる（表示崩れ） | `・` や `①②③` で通常テキストとして書く |
| `1. 番号付きリスト` | バグる | 同上 |
| `---` や `───` 区切り線 | 変換されない or 表示崩れ | 使わない。見出しで区切る |
| `# h1見出し` | noteは`##`が最大。h1は使わない | `##` を使う |
| `#### h4以下` | 変換されない | `###` までにする |
| `` `インラインコード` `` | そのまま表示される | 使わない |
| ```` ```コードブロック``` ```` | 変換されない | プレーンテキストとして書く |
| `[リンク](URL)` | 不安定 | URLを直書き |
| `![画像](path)` | 変換されない | スクリプトで挿入 |

## 2. 太字が壊れるパターンと安全パターン

noteのパーサーは太字の `**` 周辺の文字に敏感。ここが一番ハマりやすい罠。

### 壊れるパターン

```
❌ **「テキスト」**こと
   → 閉じ**の直後に全角文字がくっつくと太字にならず、**がそのまま表示される

❌ テキスト**太字テキスト**テキスト
   → **の前後にスペースも改行もないと壊れることがある
```

### 安全パターン

```
✅ **「テキスト」こと** です
   → 太字の中に「こと」まで含め、閉じ**の後にスペースか句読点を置く

✅ テキスト **太字テキスト** テキスト
   → **の前後に半角スペースを入れる

✅ **太字テキスト**
   → 行末で閉じるなら問題なし

✅ **太字テキスト** 。次のテキスト
   → 閉じ**の後にスペース＋句読点なら安全
```

### ルール（これだけ守ればOK）

1. **太字の閉じ`**`の直後に全角文字を直接くっつけない**
2. 閉じ`**`の後には必ず「半角スペース」「句読点」「改行」のいずれかを置く
3. 太字の開き`**`の前にも半角スペースを入れるのが安全
4. 迷ったら `**太字部分** です` のように書く
5. `**「` のように太字の直後に「を置かない。`「**テキスト**」` のようにカッコの内側に太字を置く
6. HTMLコメント `<!-- -->` が残っていないか確認する（noteにそのまま表示される）

## 3. 引用ボックス `>` のルール

noteの引用は **1行完結が必須**。引用内で改行すると別々のボックスに分割される。

```
❌ ダメな例：
> ・項目1
> ・項目2
> ・項目3
→ 3つの別々のグレーボックスになる

✅ 正しい例：
> この一言で全てが変わりました

ポイントは3つあります

・まず〇〇をする
・次に〇〇を意識する
・最後に〇〇で締める
```

**使い方の指針：**
- 引用は「1つの短い印象的な文」にだけ使う
- 1つの小見出しセクションにつき3〜4個程度
- 引用内に箇条書きや改行は絶対に入れない

## 4. マーカー導入文のルール

画像挿入位置の目印として、本文中に**マーカー導入文**（読者に自然な一言）を配置する。
`[[FIG_C_XX]]` のようなプレースホルダーは使わない（noteのProseMirrorに貼り付けた後、テキストノード分割で検索できなくなる問題があるため）。

### ルール

・本文の流れに合った自然な一文にすること（読者に不自然に見えない）
・記事内でユニークであること（他の箇所と被らない）
・40文字以内が望ましい（Playwrightのテキスト検索精度のため）
・鍵かっこ「」や特殊記号を含まない方が安全
・マーカーは単独行で配置し、前後に改行を入れること

### 具体例

| マーカー導入文 | 図解内容 |
|--------------|----------|
| まずは全体のロードマップを見てね | コンテンツ全体MAP |
| ポイントをまとめるとこう | ルールまとめ図 |
| 見分け方はここがカギ | 比較図 |
| 整理するとこんな感じ | パターン解読図 |

## 5. 投稿スクリプトの使い方

### ベーススクリプト

`note_tools/AIキット_note自動貼り付け.py` をベースにカスタマイズする。

### カスタマイズ箇所

#### ① ログイン情報

```python
NOTE_ID = "xxxxxxxxxx"       # ← noteのログインID
NOTE_PASSWORD = "xxxxxxxxxx" # ← noteのパスワード
```

初期設定シート（コンセプト設計ルール ■1）で取得済みの値を使う。

#### ② IMAGE_MAP（マーカー導入文 → 画像パスの対応表）

```python
IMAGE_MAP = [
    {
        "anchor": "まずは全体のロードマップを見てね",  # ← マーカー導入文
        "image": str(DIAGRAMS / "01_roadmap.png"),     # ← 画像ファイルパス
        "desc": "01: 全体ロードマップ",                 # ← 説明（ログ用）
    },
    # ... レター3個＋コンテンツ10〜11個＝計13〜14個分を設定
]
```

**anchor の値は、コンテンツ本文中のマーカー導入文と完全一致させること。**
40文字を超える場合はスクリプト内で先頭40文字に自動切り詰めされるが、できるだけ短くする。

#### ③ CONTENT_FILE（投稿するMDファイルのパス）

```python
CONTENT_FILE = BASE / "【note投稿用】〇〇_完成版.md"
```

レターとコンテンツを結合した完成版MDファイルのパスを指定する。

### スクリプトの処理フロー

1. MDファイル読み込み → タイトル抽出 → 本文取得
2. noteにログイン
3. 新規記事ページを開く
4. タイトルを入力
5. 本文テキストをクリップボード経由で一括ペースト（noteのProseMirrorが自動でリッチテキスト変換）
6. IMAGE_MAPの各アンカーテキストを順番に処理：
   - Playwrightの `text=` ロケーターでアンカーを検索
   - `scroll_into_view_if_needed()` で画面外でもスクロール
   - 行末に移動 → Enter → PowerShellでクリップボードに画像コピー → Ctrl+V
7. 全画像挿入完了 → ブラウザを開いたまま待機（手動で確認・公開）

## 6. note投稿用MDファイルの作り方

### STEP1：レターとコンテンツを結合する

```
{アカウント名}/letter.md の内容
（→ ここがnoteの無料エリア）

---

{アカウント名}/content.md の内容
（→ ここがnoteの有料エリア）
```

結合したファイルは `note_tools/【note投稿用】〇〇_完成版.md` として保存する。

### STEP2：太字・引用・記法のチェック

上記セクション1〜3のルールに違反する箇所がないか全てチェックする。

### STEP3：マーカー導入文の確認

- マーカー導入文が全て記事内でユニークか
- 40文字以内か
- 特殊記号が入っていないか

### STEP4：IMAGE_MAPの設定

Phase 2で生成した図解画像のパスとマーカー導入文の対応をスクリプトに設定する。

### STEP5：スクリプト実行

```
python note_tools/AIキット_note自動貼り付け.py
```

### STEP6：確認・公開

ブラウザ上でプレビュー確認 → 問題なければ公開。

## 7. チェックリスト（公開前に確認）

- [ ] `**太字**` の閉じ`**`直後に全角文字がくっついている箇所はないか
- [ ] `> 引用` は全て1行完結になっているか
- [ ] `- * 1.` のMarkdownリスト記法が混入していないか
- [ ] `---` や `───` の区切り線が残っていないか
- [ ] 見出しは `##` と `###` のみか（`#` `####` は使っていないか）
- [ ] HTMLコメント `<!-- -->` が残っていないか
- [ ] 4つ以上の項目が均等に並んでいる箇所はないか
- [ ] 太字が小見出しあたり5〜6個入っているか
- [ ] 引用が小見出しあたり3〜4個入っているか
- [ ] 4文以上続いている段落はないか
- [ ] 1文ごとの改行になっている箇所はないか
- [ ] マーカー導入文が全て記事内でユニークか
- [ ] IMAGE_MAPのanchorとマーカー導入文が一致しているか

## 8. よくあるトラブルと対処法

| 症状 | 原因 | 対処法 |
|------|------|--------|
| `**` がそのまま表示される | 太字の閉じ`**`直後に全角文字 | `**` の後にスペース or 句読点を入れる |
| 引用が複数ボックスに分裂 | 引用内で改行している | 1行完結にする |
| リストがおかしくなる | `- * 1.` のMarkdown記法 | `・` `①②③` で書き直す |
| 区切り線が表示されない | `---` は非対応 | `##` 見出しで区切る |
| 画像が入らない | アンカーテキストが記事内で見つからない | マーカー導入文とIMAGE_MAPのanchorを完全一致させる |
| 全体的にスカスカ | 1文ごとに改行している | 2〜3文で1段落にまとめる |
| AI感がすごい | 均等なリスト、同じ語尾の繰り返し | リストは3つまで＋分量バラバラ、語尾を崩す |
